📋 개발 지침 · Recipe Detail

1. 영상 히어로 — 롱폼 vs 숏폼

롱폼 (16:9)

• 영상 영역 position: sticky; top: {header높이}px로 고정
• 스크롤 시 헤더 아래에 붙어 콘텐츠 위에 계속 노출
• 유저가 영상 보면서 레시피를 읽을 수 있는 구조

숏폼/릴스 (9:16 원본 → 1:1 섹션 내 3:4 표시)

• 스티키 해제, 일반 플로우
• 섹션 컨테이너: aspect-ratio: 1/1; background: #1a1a1a
• 영상 이미지: aspect-ratio: 3/4; height: 100% — 1:1 컨테이너 중앙 배치
• 좌우 여백은 다크(#1a1a1a) 필러박싱으로 처리
• 숏폼은 세로가 길어 스티키 시 콘텐츠 영역을 과도하게 가림 → 의도적 해제

⚠ 영상 소스의 원본 비율(16:9 or 9:16)은 API/DB에서 전달. 프론트는 비율 값에 따라 sticky 여부를 자동 분기해야 함.

Instagram 소스

• 스티키 해제, 일반 플로우
• Instagram oEmbed 위젯을 히어로 영역에 직접 로드 (자체 영상 호스팅 아님)
• 인스타 embed UI(프로필 헤더·재생버튼)가 그대로 노출됨
• 탭 시 인스타 앱 또는 웹에서 영상 재생
• 소스 링크: "크리에이터명 · Instagram에서 보기" (YouTube와 동일 패턴, 플랫폼명만 교체)
• 타임스탬프 인터랙션 미지원 (인스타 임베드는 외부 seek 불가)

⚠ 소스 플랫폼(youtube / instagram)은 API/DB에서 전달. 프론트는 플랫폼 값에 따라 히어로 컴포넌트를 분기.
⚠ Instagram oEmbed API 호출 시 access token 필요 (Meta Developer App 필수).

직접업로드 (크리에이터 자체 업로드)

• 스티키 해제, 일반 플로우
• 커버 이미지: aspect-ratio: 1/1 정방형 (영상 아님, 정적 이미지)
• 소스 링크: "졸리팟 오리지널 레시피" (외부 링크 없음)
• 타임스탬프 뱃지 미표시 (영상이 없으므로 seek 불가)
• 만드는 법 각 단계에 스텝 이미지 표시 (4:3 비율, 스텝 텍스트 상단 배치)
• 플레이 버튼 없음

⚠ 소스 타입(youtube / instagram / direct)에 따라 히어로·스텝 이미지·타임스탬프 표시를 분기해야 함.

2. 만드는 법 — 타임스탬프 인터랙션

• 각 스텝의 파란색 타임스탬프 뱃지 00"05를 누르면 영상의 해당 장면으로 이동(seek)
• 자동재생은 하지 않음 — seek만 수행, 재생은 유저가 직접 플레이 버튼을 눌러야 함
• 롱폼(스티키) 상태에서 탭하면 상단 고정된 영상이 해당 위치로 즉시 이동 → 유저가 영상+텍스트를 동시에 확인 가능
• 숏폼(비스티키) 상태에서 탭하면 영상 영역으로 스크롤 이동 후 seek

⚠ 타임스탬프 값은 초(seconds) 단위로 저장. 표시 포맷은 MM"SS로 변환 (예: 65초 → 01"05). 탭 영역은 뱃지 전체.

3-2. 직접업로드 소스 (본인 vs 타인)

영역 타인 시점 본인 시점
히어로 1:1 정방형 커버 이미지 (영상 없음, 플레이 버튼 없음)
제목 아래 소스 링크 없음
크리에이터 카드 아바타 + 이름 + 팔로우 버튼 "내 레시피" 라벨 + "편집" 버튼
스텝 이미지 있음 (4:3 비율, 스텝 텍스트 상단 배치)
타임스탬프 없음 (영상이 없으므로)
재료 이하 동일

⚠ 분기 기준 필드: source_type(YOUTUBE / INSTAGRAM / DIRECT) + recipe_type(AI_SUMMARY / ORIGINAL) + is_owner(boolean)
⚠ 직접업로드는 항상 오리지널 — AI 요약 Case 불가.
⚠ 오리지널 + 본인 시점: 크리에이터 카드에 "팔로우" 대신 "내 레시피" 라벨 + "편집" 버튼 표시. 오리지널도 크리에이터 등록이 전제이므로 본인/타인 분기 필수.
⚠ AI 요약은 외부 소스 큐레이션이므로 본인/타인 분기 없음 — is_owner 무시.
⚠ 크리에이터 카드·알림받기·큐레이터 바는 AI 요약에서 완전히 제거된 상태(DEC-022/023).

4. 인분 조절 (Serving Adjuster)

• "재료" 섹션 헤더 우측에 −/+ 원형 스텝퍼 배치
• 메타 행의 "1인분"은 정적 텍스트로 유지 — 레시피 기본 인분 표시 전용
• 범위: 1~10인분. 스텝퍼로 조절
• 인분 변경 시 재료 테이블의 모든 수량이 비례 자동 스케일링
• 기본 인분(base_serving)은 API/DB에서 전달. 현재 인분 ≠ 기본 인분일 때 카운트 텍스트를 테라코타로 강조하여 "수정된 상태"임을 인지
• 소수점: 1자리까지 표시, 정수는 소수점 생략 (예: 1.5g, 100ml)

⚠ 재료 데이터에 수량(numeric)과 단위(unit)가 분리 저장되어야 스케일링 가능. "적당량", "약간" 등 비정량 재료는 스케일링 대상 제외 — 해당 셀은 고정 텍스트.
⚠ 메타 행의 정적 "1인분" 표시와 재료 헤더의 인분 스텝퍼를 혼동하지 않도록 주의. 메타 행은 원본 레시피의 기준 인분, 스텝퍼는 유저가 조절한 현재 인분.
⚠ 칼로리(452kcal)는 인분 조절에 연동하지 않음 — 메타의 칼로리는 1인분 기준 고정값.

5. 재료 테이블 + 쿠팡 구매 동선

• 2열 그리드: 각 셀 = 재료명 (용량) 왼쪽 정렬 + 🛒 아이콘 오른쪽 정렬
• 좌우 열 사이 세로 구분선 1px, 행 구분선 없음
• 🛒 아이콘 = 테라코타(opacity 0.85), 26px — "있되 과하지 않게"
• 아이콘 탭 시 해당 재료의 쿠팡 파트너스 개별 상품/검색결과 링크로 이동

재료명 링크 방식 폐기 (DEC 기록)

• 재료명 자체에 링크를 거는 방식은 폐기. 이유: ① 밑줄 없는 색상만으로는 클릭 가능 인지가 약함(WCAG 1.4.1 색상 의존) ② 8개 재료명이 링크화되면 정보 가독성 저하 ③ 재료명=정보(차콜)/구매=액션(테라코타) 역할 분리 원칙 위배
• 결론: 정보(재료명)와 액션(🛒)을 분리. 아이콘이 비색상 단서 역할도 하여 접근성 우위

⚠ 쿠팡 파트너스는 묶음/일괄 장바구니 담기 미지원. "재료 한 번에 담기" 불가 — 재료별 개별 링크만 가능.
⚠ 모든 링크는 유저 명시적 탭. 자동 실행·플로팅 배너·스치기 진입은 쿠팡 운영정책 위반(수익 몰수+계정 해지) — 절대 금지.
⚠ 가격·평점 등은 쿠팡 파트너스 API 제공 범위 내에서만 표시. 크롤링 데이터 사용 금지.

6. 장보기 (레시피로 장보기)

• 횡스크롤 상품카드, overflow-x: auto + 스크롤바 숨김
• 쿠팡 파트너스 Disclaimer는 시스템 자동 표시 (하드코딩 아님)
• 재료 영역(🛒 아이콘)과 장보기 섹션 모두 쿠팡 연결 — 재료는 개별 링크, 장보기는 큐레이션 상품카드

7. 요리 시작 버튼 (이번 버전에서 제외)

⚠ 이번 버전에서는 "요리 시작 / 쿡모드 진입" 버튼을 의도적으로 제외. "만드는 법"은 섹션 헤더만 단독 노출하며, 단계 리스트는 항상 펼쳐진 상태로 제공.

8. 이런 레시피는 어때요 — 무한스크롤

• 3열 균일 그리드, aspect-ratio: 3/4
• 썸네일 풀블리드 + 하단 그라데이션 오버레이(레시피명 + 조회수 + 소요시간)
• 무한스크롤 페이지네이션 — 하단 도달 시 다음 배치 로드
• 카드 탭 시 해당 레시피 상세로 이동

10. 하단 엔게이지먼트 바 (컬렉션·레시피 통일)

• 콘텐츠 상세 화면(컬렉션·레시피)은 5탭 네비 대신 동일한 엔게이지먼트 바 사용
• 구조: [댓글 남기기...] [♡ N] [🔖 N] [💬 N] [↗]
• position: fixed; bottom: 0 + padding-bottom: max(10px, env(safe-area-inset-bottom))

댓글 입력 필드

• 좌측 pill 인풋 (크림 배경 #F5F0E8, 보더 #EDE8E0, h36, r18)
• 탭 시 댓글 섹션으로 스크롤 이동 또는 키보드 올림

엔게이지먼트 아이콘

액션 비활성 활성
♡ 좋아요 stroke #6B6B6B fill+stroke #C75D3E
🔖 저장 stroke #6B6B6B fill+stroke #2A2A2A
💬 댓글 카운트만 표시 (토글 없음)
↗ 공유 시스템 공유 시트 호출

⚠ 헤더의 ← 뒤로가기가 유일한 탈출 경로. 헤더 sticky 유지 필수.
⚠ 컬렉션·레시피 하단바는 동일 컴포넌트로 구현하여 일관성 유지.

10-1. 엔게이지먼트 카운트 노출 임계값

부정적 사회증명 방지를 위해 카운트가 일정 수 미만이면 숫자를 숨기고 아이콘만 표시한다.

액션 임계값 미만 시
♡ 좋아요 10 아이콘만 (숫자 숨김)
🔖 저장 5 아이콘만 (숫자 숨김)
💬 댓글 3 아이콘만 (숫자 숨김)

⚠ 본인 시점 + count=0이면 항상 숫자 숨김(아이콘만).
⚠ 임계값은 출시 초기 운영 파라미터. 데이터 누적 후 재검토 예정.
⚠ 이 규칙은 하단 엔게이지먼트 바뿐 아니라 콘텐츠 상세 전체(레시피·컬렉션·요리후기)에 공통 적용.

10-2. 후기 0건 처리

부정적 사회증명 방지. 후기가 없을 때 "0명이 만들었어요"를 보여주면 역효과.

영역 별점 있음 별점 없음 (신규)
메타 ⭐ 별점 4번째 셀: ⭐ 4.8 4번째 셀 유지 → ✨ "신규" 배지로 대체 (셀 숨기지 않아 레이아웃 흔들림 없음)
포토 갤러리 "이 레시피로 만든 요리 24" + 사진 횡스크롤 섹션·제목 유지, 카운트·전체보기 숨김, 본문은 빈 상태 안내("아직 만든 사람이 없어요 · 첫 번째로 만들고 후기를 남겨보세요")

⚠ 상단 프로토타입 토글의 "별점 있음 / 별점 없음 (신규)"으로 두 상태를 전환해 확인. 별점·후기·포토는 신규 레시피에서 함께 0건이 되는 한 묶음으로 처리.

⚠ 후기 카운트 = CookHistory 기반 실행 후기 수. 댓글(일반 의견)은 포함하지 않음.

10-3. 이 레시피로 만든 요리 (포토 갤러리)

• 위치: 만드는 법(AI 라벨) 아래, 댓글 위
• 구조: 헤더 "이 레시피로 만든 요리 N" + "전체보기 ›" + 정사각 사진 횡스크롤
• 사진(100×100) 탭 시 해당 요리후기 상세로 이동 (별점·신뢰칩·후기 텍스트는 후기 상세에서 표시)
• 데이터 소스: CookHistory(불변 기록) 기반. 쿡모드 완료 시 생성된 후기 사진
• 메타 행 ⭐ 별점 요약과 한 쌍 — 별점은 평균, 갤러리는 증거 사진

레시피 상세는 진입점만, 후기 본문은 전용 화면

• 레시피 상세에서 후기 텍스트를 펼치지 않음 — 사진 횡스크롤로 "N명이 만들었다"는 증거만 노출
• instruction 집중: 레시피 페이지는 "만들지" 결정을 돕고, 후기 탐색은 전용 화면으로 분리

⚠ 후기 0건 처리: 메타 ⭐ 셀 숨김(3열만) + 갤러리 섹션 숨김. 단, 댓글↔추천 구분선은 유지.
⚠ 후기 카운트 = CookHistory 기반 실행 후기 수. 댓글(일반 의견)은 포함하지 않음.

10-4. 댓글 (진입점 + 전체화면)

레시피 상세 내 댓글 진입점

• 헤더: "댓글 N" + "전체보기 ›" (갤러리 헤더와 동일 패턴·위계로 통일)
• 미리보기: 최근 댓글 2개 (아바타 24px + 이름 + 본문 1줄 clamp)
• 입력칸: 미리보기 아래 "나 아바타 + 댓글 남기기..." 필드
• 진입점·미리보기·입력칸·하단바 💬·하단바 입력 — 모든 경로가 댓글 전체화면으로 이동

댓글 전체화면 (오늘의집 패턴)

• 별도 풀스크린 (인라인 아님). 헤더 "← 댓글 N"
• 댓글 리스트: 아바타 + 이름 + 날짜 + 본문 + "답글 달기"
• 대댓글(답글): 좌측 들여쓰기(padding-left 38px) + @멘션(info blue)
• 하단 고정 입력창 + 전송 버튼(테라코타 원형)
• ← 뒤로가기로 레시피 상세 복귀

댓글 vs 요리후기 — 성격 구분

• 댓글 = 질문/의견 ("생크림 대신 뭐 써도 돼요?"), 하단바·진입점에서 작성
• 요리후기 = 실행 증언 (별점+신뢰칩+사진), 쿡모드 완료 플로우에서만 작성
• 두 데이터·생성 경로는 분리 유지. 단 레시피 상세 UI에서는 후기=사진 갤러리, 댓글=텍스트 진입점으로 각각 노출

⚠ 댓글 카운트는 대댓글 포함 여부를 기획에서 확정 필요 (현재 프로토는 최상위 댓글 수 기준).
⚠ 댓글 0건일 때: 미리보기 영역 숨김, "첫 댓글을 남겨보세요" 입력칸만 노출.

10-5. 에러 / 빈 상태 (졸리팟 핵심)

관통 원칙: 졸리팟의 본체는 영상이 아니라 레시피 카드. "정확도=존재 이유"이므로, 불확실할 땐 숨기거나 정직하게 경고하되 틀린 정보를 확신에 차서 보여주지 않는다. 기술 코드(500, ECONNREFUSED 등) 절대 노출 금지.

① 영상 삭제 / 비공개 — 우아한 강등

• 영상 영역만 플레이스홀더로 대체 ("원본 영상을 더 이상 볼 수 없어요")
• 레시피 본문(재료·만드는 법·댓글)은 완전 정상 노출 — 전체 페이지 에러 처리 금지
• 소스 링크("YouTube에서 보기") 숨김 (깨진 링크 방지)
• 타임스탬프 뱃지 비활성 (seek 대상 없음)

② 네트워크 에러

• 진입 시 전체 실패: 전체 빈 상태 ("레시피를 불러오지 못했어요 / 인터넷 연결을 확인해 주세요" + [다시 시도])
• 연결 복구 감지 시 자동 재시도 (유저 액션 전)
• 부분 실패(이미지·상품만): 해당 영역만 플레이스홀더 + 재시도, 레시피 본문은 유지 — 전체 덮지 말 것
• 타임아웃(10~15초): 스피너 멈추고 "취소 / 계속 기다리기" 옵션 제공

③ 추출 실패 / 저신뢰

• 저신뢰(low/needs_review): 경고 배너 노출 ("AI가 자동으로 정리한 초안이에요") + 원본 영상 보기 강조 + 신고 버튼. 차단하지 않되 신뢰 수준 명시
• 추출 완전 실패: 부분 깨진 레시피 대신 빈 상태 ("아직 레시피로 정리하지 못했어요") + [원본 영상 보기]로 안전하게 이탈 유도
• 추출 진행 중: 일반 스피너 ❌ → 스켈레톤 UI (재료·스텝 자리 회색 블록) + "AI가 영상에서 레시피를 정리하고 있어요"

케이스 핵심 원칙 금지
영상 삭제 레시피는 살린다 (graceful degradation) 전체 페이지 에러 처리
네트워크 부분 실패는 부분만, 자동 재시도 기술 코드 노출, 전체 덮기
저신뢰 추출 정직하게 불확실성 노출 + 원본 fallback 틀린 걸 확신에 차서 보여주기

⚠ 저신뢰 라벨은 일반 AI Disclaimer(만드는 법 하단)와 별개 — 재료 위에 강조 배너로 별도 노출.
⚠ confidence 값(high/low/needs_review)은 추출 파이프라인에서 전달. high=자동게시(일반 라벨), low=경고 배너.

11. 스타일 체크리스트

• 순검정 #000 사용 금지 → 모든 텍스트 #2A2A2A (차콜)
• font-weight: 700은 섹션 헤더(L1·L2)와 가격에만 허용. 그 외는 500 이하
• 카드 컴포넌트: background: #F5F0E8 + border: 1px solid #EDE8E0 — 보더 누락 금지 (OLED 대응)
• 그림자 사용 안 함 — 전체 플랫 디자인, 보더로 경계 구분
• 서체: Pretendard (시스템 폰트 폴백: -apple-system, Apple SD Gothic Neo)

12. 프로토타입 전용 요소 (개발 미포함)

• 상단 고정 토글 바 (Case 1/2 전환, 롱폼/숏폼 전환) — 프로토 검토용
• 이모지 플레이스홀더 (썸네일·상품 이미지 자리) — 실 이미지로 교체
• 그라데이션 배경의 히어로 영역 — 실제 영상 플레이어로 교체
