v11-1 — 완성 직후 커머스 제거

변경

  • ③ 등록 후 화면에서 "재료 다시 채우기"(커머스) 제거
  • 디스커버리("이런 레시피는 어때요?")만 유지

왜 제거

  • 문구 문제: 방금 요리 완성했는데 "재료 다시 채우기"는 타이밍 어색 (재구매는 다음에 또 만들 때)
  • 감정 문제: 완성 = 감정 정점인데 "상품 사세요"가 분위기 깸. "뿌듯하다" 상태에 판매 끼어듦
  • 완성 직후 자연스러운 행동은 "다음엔 뭘 만들까"(디스커버리)이지 "상품 구매"가 아님

커머스 funnel은 잃지 않음

  • 커머스의 올바른 자리 = 구매 의도가 자연 발생하는 맥락
  • 레시피 상세의 "이 레시피로 장보기" (요리 시작 전, 재료 필요할 때)
  • ③ 디스커버리 → 레시피 상세 → 장보기로 funnel은 살아있음
  • 완성 직후에 억지로 끼우지 않을 뿐 — 제자리로 보낸 것

③ 최종 구성

  • 확인 (세이지 체크 + "후기 고마워요")
  • 이런 레시피는 어때요? (2열 그리드)
  • 홈으로

v11 — 후기 등록 후 화면 (funnel 회수)

왜 이 화면이 필요했나

  • v7~v10에서 후기 작성에 집중하면서 v6가 가졌던 커머스·디스커버리 funnel 상실
  • 완성 직후는 "재료 또 사고 싶은" 최적 타이밍 — 이걸 못 살리면 반쪽
  • 해법: 후기 등록 후 화면에서 회수 (작성과 분리해 인지 부담 없이)

3-Phase 전체 흐름

  • ① 진행 중: 스텝 진행 (인라인 칩·타이머·팁)
  • ② 완성: 감정 정점 (confetti + 마스코트) + 후기 작성 통합 카드
  • ③ 등록 후: 차분한 확인 + 커머스 + 디스커버리

③ 화면 구성

  • 확인 (차분): 세이지 체크 + "후기 고마워요" — ②에서 이미 큰 축하했으니 가볍게
  • 재료 다시 채우기: 가로 상품 스크롤 (커머스 회수) + 파트너스 고지
  • 이런 레시피는 어때요?: 2열 그리드 (디스커버리 회수)
  • 홈으로: secondary 버튼

감정 곡선 설계

  • ① 집중 → ② 정점(축하·후기) → ③ 차분한 정리·다음 행동
  • ②에서 큰 축하를 했으므로 ③는 톤 다운 — confetti·큰 hero 없음
  • ③의 메시지는 등록 여부로 동적: 등록="후기 고마워요" / skip="스크랩북에 기록됐어요"

측정 포인트

  • ③ 도달률 (후기 등록 + skip 모두 도달)
  • "재료 다시 채우기" 클릭률 (완성 직후 커머스 전환)
  • "비슷한 레시피" 클릭률 (다음 세션 트리거)
  • ③에서 홈 이탈 vs 추가 액션 비율

v10 — 별점+후기+사진 통합 카드

변경

  • 별점 섹션 + 후기 섹션 → 하나의 카드로 통합
  • 카드 제목: "이번 요리, 어땠어요?" → "만들어보니 어땠나요?"
  • 제목 하나가 별점·후기·사진 전체를 포괄 (질문 중복 제거)
  • 카드 구조: 제목 → 별점 → divider → textarea → 사진 옵션
  • 등록·skip 버튼은 카드 밖

왜 한 카드로

  • 별점은 미끼, 후기가 본질 — 별점 1탭이 "이미 시작했다"는 관성을 만들어 후기까지 이어짐
  • 순차 분리 시 별점 주고 "끝났다" 인지 → 후기 안 씀. 한 덩어리여야 "아직 안 끝났네"
  • 제목 1개 + 별점 + 후기 + 사진이 시각적으로 "하나의 행동 묶음"임을 전달
  • focus-within으로 카드 전체 border 강조 → 입력 중 통일감

문구 결정

  • "후기" (사무적/리뷰 느낌) → "만들어보니 어땠나요?" (대화체, 따뜻함)
  • 졸리팟 톤(따뜻한 살림)에 맞는 질문형 — 평가 부담 ↓
  • placeholder: "느낀 점을 자유롭게 남겨보세요... (선택)" — 강요 X

최종 close phase 흐름

  • Hero (마스코트 + confetti + 완성! + 27번째 요리)
  • 통합 카드: 만들어보니 어땠나요? + ★ + 후기 + 📷
  • 후기 남기기 / 나중에 남길게요

v9 — 사진을 textarea에 통합

변경

  • 별도 큰 사진 영역 (가로 88px 박스) 폐기
  • textarea + 사진 옵션을 하나의 컨테이너로 통합
  • 사진은 textarea 하단 inline 버튼 — "사진 추가" (작은 텍스트)
  • 사진 추가 시 같은 위치에 작은 썸네일 + 제거 ✕ 버튼
  • focus-within으로 컨테이너 전체 border가 terracotta로 강조

왜 사진을 통합했나

  • 사진은 비용 ↑ (카메라/갤러리 권한, 탐색, 선택 등 5-6탭) — 별점·후기보다 무거움
  • 별도 큰 영역으로 두면 "사진까지 찍어야 하나?" 부담 → 후기 작성률 자체가 떨어짐
  • 인스타·카톡·트위터 등 검증된 패턴: 텍스트 입력 옆 작은 첨부 아이콘
  • 사진 = 후기의 부수 요소, 본문은 텍스트라는 위계 명확화

점진적 깊이 (Progressive Disclosure)

  • 1단계: 별점만 — 1탭, 가장 쉬움
  • 2단계: 후기 텍스트 — 보통, 메인 액션
  • 3단계: 사진 — 선택, 부수 첨부
  • 각 단계는 독립적으로 완료 가능 — 어떤 조합으로도 등록 가능

측정 포인트

  • 별점 전환률 / 후기 전환률 / 사진 첨부률 (3단계 funnel)
  • 후기 평균 글자 수 (engagement)
  • 사진 포함 후기 비율 (UGC 품질)
  • v7-1(별도 사진 영역) vs v9(통합) A/B 가능

v8 — Active 마지막 → Close 자동 전환

변경

  • cook-mode 내 별도 complete-screen 영역 제거 (어색한 "완성 사진 올리기 / 처음으로" 버튼)
  • 마지막 step의 "요리 완성" 버튼 → switchPhase('close')로 view 자동 전환
  • 두 phase가 자연 연결되는 단일 흐름 형성
  • ② 완성 탭은 그대로 유지 (편의용 데모 진입점)

왜

  • v4 시절 complete-screen은 별점 + 사진 업로드 + 처음으로 — 임시 디자인이었음
  • v7에서 새 close phase 디자인을 만들어 두고, v4 잔재가 active 안에 남아있는 게 모순
  • 같은 "요리 완성" 액션이 두 곳에 중복 → 일관성 ↓
  • 마지막 step → close 자동 전환으로 cook 전체 흐름이 하나로 통합

흐름 변화

  • 이전: step 1 ~ 7 → "요리 완성" 버튼 → cook-mode 내부 complete-screen (별점·사진·재시작)
  • 이후: step 1 ~ 7 → "요리 완성" 버튼 → close phase 자동 전환 (마스코트·confetti·별점·사진·후기)

검증 흐름

  • ① 진행 중 탭에서 "다음 단계" 6번 → 마지막 step (완성)
  • "요리 완성" 버튼 클릭 → close view로 자동 전환 + confetti 트리거
  • 또는 ② 완성 탭 직접 클릭 → 같은 close view

v7-1 — "26분 소요" 제거

왜 제거

  • Cook mode 진입~종료 wall clock time은 측정 가능
  • 하지만 일시정지·백그라운드·중간 이탈 등으로 실제 요리 시간을 대표 못 함
  • "26분 안 걸렸는데?" 같은 사용자 의심 → 신뢰도 ↓
  • 부정확한 데이터 노출보다 안 보여주는 게 안전

대신 남긴 것

  • 27번째 요리 — 단순 누적 카운터, 정확 측정 가능
  • 사용자의 누적 성취감 표현 (가벼운 게이미피케이션)
  • 아이콘 시계 → award 아이콘 (성취 메타포로 변경)

향후 (P2+)

  • "이 레시피 N번째 도전" (레시피별 카운터) — 재요리 패턴 측정
  • 실제 요리 시간 측정 가능해지면 (백그라운드 제외 등 정교한 추적) 재추가 검토

v7 — 후기 작성 단일 흐름

변경

  • 액션 카드 5개 (댓글·사진·스크랩북·장보기·비슷한레시피) 전부 폐기
  • 대신 한 화면에서 별점 → 사진 → 후기 텍스트 → 등록 단일 흐름
  • 댓글은 별도 분기가 아니라 close phase의 default action으로 통합
  • "나중에 남길게요" skip 옵션은 유지 (강요 X 원칙)

왜 단일 흐름인가

  • 완성 직후는 가장 긍정 감정 모먼트 — 액션 분기보다 한 곳에 집중시키는 게 conversion 강함
  • 5개 카드 = 5개 선택 부담. 사용자는 "뭘 먼저 누를까" 고민 → 결국 아무것도 안 함
  • 댓글 작성은 핵심 SNS funnel — 별도 화면 이동 없이 즉시 작성 가능하게 마찰 ↓
  • 사진+텍스트가 한 화면에 같이 있어 "사진 있는 후기" 작성률 ↑ (UGC 품질 향상)

빠진 액션들은 어디로

  • 스크랩북 저장: 자동 저장이라 별도 알림 불필요 (P2에서 작은 토스트로 가능)
  • 재료 다시 채우기: 후기 등록 완료 후 "다음에 또" 화면(P2)에서 노출
  • 비슷한 레시피: 후기 등록 완료 후 같은 위치에서 노출
  • 핵심: 후기 작성 완료 = 새 디스커버리·커머스 진입점 트리거

입력 영역 설계

  • 별점: 그대로 5점 별 (compact)
  • 사진: 가로 88px 박스 + dashed border. "선택" 라벨로 강요 X
  • 후기 텍스트: textarea 110px, focus 시 terracotta border. placeholder가 안내 역할
  • 버튼: 풀폭 56px 테라코타 "후기 남기기" — 모든 입력 비어도 활성 (별점만 있어도 OK)
  • Skip: 작은 링크 "나중에 남길게요"

측정 포인트

  • 후기 등록률 (총 close 도달 / 후기 submit)
  • 사진 포함 후기 비율 (UGC 품질)
  • 후기 평균 글자 수 (engagement 깊이)
  • Skip 비율 (강요 톤 검증)

v6 — Close phase 전면 재설계

몰입도 관점 5원칙 적용

  • Emotional peak first — Hero 화면 55% + confetti 3.5초로 감정 정점 먼저
  • Visual dominance — Hero 압도적 비주얼, 액션은 그 다음
  • Brand moment — 노른자 캐릭터 SVG + 노른자 그라데이션 (완성은 브랜드 IP 모먼트)
  • Real over abstract — CSS-drawn egg 폐기 → 정성스러운 SVG 마스코트 (eyes·cheeks·smile·highlight)
  • Compress actions — 사진 캡처도 액션 카드로 통합 → 위계 정돈

핵심 변경

  • 배경: deep beige(#EAE0CD, 탁함) → very light cream(#FBF8F2)
  • Hero: 24px → 36px padding, 노른자 그라데이션 강화 (radial 9개 색 stop)
  • 마스코트: CSS pseudo-elements → SVG 일러스트 (yolk gradient, cheeks blush, eye highlight, round smile)
  • Confetti: 22개 컨페티가 4컬러로 떨어짐 (노른자·테라코타·세이지·gold). 자동 3.5초 fade-out
  • "완성!": 28px → 36px (압도적 위계)
  • 사진 캡처: 독립 4:3 영역 → 액션 카드 1줄로 통합 (강요 X 톤 유지)
  • 액션 카드: 4개 → 5개 (사진 통합)
  • Confetti 트리거: ② 완성 탭 클릭 시 자동 실행

왜 사진 캡처를 카드로 통합

  • 독립 큰 영역은 사진 안 찍는 사람에게 dead zone
  • 액션 카드 1줄로 통합 → 위계 평준화 (사진 안 찍어도 화면 잘 보임)
  • 탭 시 카드 내부에서 "사진 추가됨 ✓"로 변경 → 인지 강화
  • 사진은 댓글 작성과 연계되니까 댓글 카드 바로 아래 배치

측정 포인트

  • Hero 노출 → 액션 클릭 전환률 (긍정 감정 효과 측정)
  • 각 액션별 클릭률 (사진·댓글·장보기·비슷한레시피)
  • 별점 분포 (P2 추천 input)
  • Confetti 유무 A/B 테스트 가능 (CSS class 토글로)

v5-1-2 — 배경 톤 + 별 진하기

변경

  • Close 페이지 배경: 크림(#F5F0E8) → deep beige (#EAE0CD)

  • 위계 반전: 페이지 크림 → 카드 흰색 (디자인 가이드 3단계 위계 유지, 톤만 반대로)
  • Hero gradient 끝점도 크림으로 자연 페이드
  • Hero-meta 칩 흰색 opacity 0.6 → 0.75 (크림 위 가시성 보정)
  • Photo-area: 크림 → 흰색 카드 + dashed border
  • Action-card: 크림 → 흰색 카드
  • Action-icon: 흰색 → 크림 (카드 안에서 강조)
  • Primary action card tint: 0.04 → 0.06 (크림 배경 위 구분 강화)

왜 크림 배경

  • "집에 있는 듯한" 따뜻한 분위기 — 완성 모먼트와 결 일치
  • Hero의 노른자 그라데이션이 페이지 배경(크림)으로 자연스럽게 흘러나옴 — 흰색일 때보다 매끄러움
  • 흰 카드가 페이지 위에 떠 보여서 액션 가독성 ↑
  • Cook mode active phase(흰색)와 시각적 대비 → "끝났다, 다른 단계다" 감각 전달

v5-1 — 자평 별점 변경

변경

  • 3-tier 이모지 (😊/😐/😅) → 5점 별점
  • 별 색: 노른자 옐로우 (#FDB813) — close phase의 브랜드 모먼트 톤과 일관
  • 별점 탭 시 1~N까지 자동 채움 + drop-shadow로 강조

왜 별점

  • 표준 메타포 — 인지 비용 없음
  • 정량 데이터 (1~5) — 평균 평점, 분포 등 활용도 ↑
  • 이미 다른 화면(레시피 카드 등)에서도 별점 사용 가능 — 시스템 일관성
  • 3-tier는 표본 부족 시 분포 한쪽 쏠림 — 5-tier가 더 분별력 있음

활용 시나리오

  • 레시피 카드에 평균 평점 노출 (예: ★ 4.3 · 87명)
  • P2 추천 알고리즘 input — 사용자가 4점 이상 준 레시피 패턴 학습
  • 레시피 신뢰도 신호 — AI 요약본 vs 오리지널의 별점 분포 비교

v5 — Active + Close 통합

구조

  • 두 phase를 같은 phone frame에 두 view로 배치
  • 상단 phase 탭으로 전환 — ① 진행 중 / ② 완성
  • 실제 앱에서는 마지막 step 종료 → 자동으로 close phase 진입

Phase 1. Active (v4.1 그대로)

  • 인라인 amount/timer 칩, 26px 본문, 64px 타이머바
  • 스텝별 조건부 팁 영역 (cream-alt + 세이지 전구)
  • 새 타이머 추가 시 우측 끝 auto-scroll

Phase 2. Close (신설)

  • Hero: 노른자 마스코트 + bounce 애니메이션 — 브랜드 IP 모먼트
  • 사진 캡처: 강요 X, "건너뛸 수 있어요"
  • 자평 5점 별점: 노른자 별, 1탭으로 N까지 채움 (v5-1 변경)
  • 다음 액션 4종: 댓글(primary) → 스크랩북 확인 → 장보기(커머스) → 비슷한 레시피
  • 가장 긍정적 감정 모먼트에서 SNS·커머스 funnel 시작

Close 액션 위계

  • 댓글이 primary (테라코타 보더) — SNS 핵심 funnel
  • 스크랩북은 이미 자동 저장 — 인지만 강화 (세이지 체크)
  • 장보기는 P1에서는 항상 노출 (커머스 임팩트 검증)
  • 비슷한 레시피는 ambient — 다음 세션 트리거

왜 PREP phase는 없는가

  • 레시피 상세 페이지(b3)가 사실상 PREP 역할
  • 재료 확인 → 장보기 → "요리해보기" 탭 = commit 모먼트
  • 별도 PREP 화면은 같은 정보 재노출 → 인지 부담 ↑
  • 대신 active 진입 시 wake lock + 짧은 toast로 화면 유지 안내

측정 지표

  • Close phase 도달률 = active 완주율 (이탈 funnel)
  • 각 액션 클릭률 (댓글·장보기·비슷한레시피)
  • 자평 별점 분포 → 레시피별 평균 평점 (P2 추천 input)
  • 완성→댓글 작성 전환률 (핵심 KPI)

요리 모드 — 개발 핵심 지침

핵심 철학

  • Cook mode는 instruction — 그 순간의 다음 한 동작만 보여준다
  • 한 화면 한 동작 원칙 — 진행·조정은 cook mode, 변형·회고·배경은 brief
  • Opportunistic UX 가정 — 사용자는 plan대로 안 간다. 그 순간 보고 즉흥 결정. 그래서 변형·대체·조정 정보는 cook mode에 있어야 작동함
  • 부엌 거리 가독성 — 화면 ~50cm + 손 자유롭지 않음. 폰트·터치 영역을 일반 모바일 UI보다 한 단계씩 키운다

컴포넌트 위계 (상→하)

  • 스텝 본문: 26px / 500 / 차콜 — 페이지 최강 위계
  • 영상 임베드: 해당 스텝 구간만 (start~end)
  • 팁 (조건부): 14px / 400 / 미드그레이 + cream-alt 배경 + 세이지 전구. 본문보다 한 단계 약하게
  • 인라인 amount 칩: 본문 내, 양은 항상 노출 (탭/팝업 금지)
  • 인라인 타이머 칩: 22px / 600 + 세이지 배경 + 흰 글씨 — 시간을 액션으로
  • Sticky 타이머바: 64px / 가로 스크롤 / 새 카드 추가 시 우측 끝 auto-scroll

데이터 모델

Step {
  title: string,
  html: string,        // amount/timer 인라인 마크업
  tip?: string | null, // 옵셔널, 거의 모두 cook_mode
  video: { start, end }
}
  • tip은 거의 모든 케이스에서 cook_mode placement
  • 사후 회고용("다음번엔...")만 brief로 분리
  • Gemini Flash 추출 파이프라인에서 steps[].tip 필드 별도 추출 (v1.4 프롬프트 룰)

인터랙션 패턴

  • 다음: 풀폭 next 버튼 (테라코타) + 스와이프 좌
  • 이전: 다음 왼쪽 48×60 prev (cream surface) + 스와이프 우. 안내 문구 X
  • 타이머 시작: 인라인 칩 탭 → 풀스크린 시트 → sticky bar 압축
  • 타이머 추가 직후: timer-bar.scrollTo(scrollWidth, smooth) — requestAnimationFrame으로 DOM 반영 후 호출

금지 사항

  • 노른자 옐로우 CTA — 브랜드 IP 전용 (디자인 가이드 §4.1)
  • 정보용 팝업/배너 — instruction-only 원칙 위반
  • 재료양 별도 탭/팝업 — 인라인 chip 원칙 위반
  • 본문(L3 26px)보다 강한 보조 정보 — 위계 무너짐
  • 사후 회고·배경·이론 — brief로 옮긴다
  • "좌우로 밀어 이전" 안내 문구 — UI 자체로 표현, 텍스트 안내 불필요
