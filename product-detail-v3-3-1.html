📋 개발 지침

상품상세페이지(PDP) v3-3 · DEC-027 · 2026.05

⭐ 핵심 원칙 (먼저 읽기)

1. 얼굴과 뒤편의 분리
졸리팟의 "얼굴"(홈·레시피·장보기)에는 레시피 연결 상품만 노출 → 신규 유저에게 "레시피 플랫폼" 정체성 유지. "뒤편"(쇼핑·검색)에서는 전 상품 접근 가능 → 기존 고객 매출 유지. PDP는 이 두 갈래를 모두 받으므로 Tier 분기가 필수.

2. Tier 1의 킥은 스타일링샷
"이 재료로 완성한 요리"가 쿠팡·네이버에 없는 졸리팟만의 가치. PDP 전환의 핵심 가설이므로 우선순위 최상. 다른 요소가 이걸 가리지 않도록 함.

3. 가격 영역 인지부하 경계
할인율·쿠폰·긴급성·적립까지가 적정 한계. 재고 임박은 "요소 과다"로 기각됨. 신규 요소 추가 시 이 영역의 밀도를 반드시 재검토.

4. 데이터로 검증할 것 (P1 출시 후)
스타일링샷의 전환 기여 · 쿠폰적용가 강조 효과 · 에디터 코멘트 가치(현재 OFF). PDP를 더 만지지 말고 데이터로 판단.

5. 기각된 안 (재논의 금지)
5버전 단일안 → Tier 분기로 수렴 / "~에서 오셨네요" 컨텍스트 배너 / 신뢰 시그널 스트립(당일발송·정품보장) / 첫뷰포트 구매수 배지(데이터 부족) / 재고 임박 / 함께둘러보면 별도 섹션. 모두 검토 후 제거 결정됨.

1. PDP 분기 로직

상품 DB에 linked_recipe_count 필드 기준으로 자동 분기.

Tier 1 (linked_recipe_count >= 1): 레시피 연결 상품. "이 재료로 완성한 요리" 스타일링샷 섹션 노출.
Tier 2 (linked_recipe_count == 0): 일반 상품. 스타일링샷 섹션 미노출, 배송 → 스티키 탭 직행.

2. 조건부 요소

타임세일 바 상품의 timesale_end_at 존재 시 노출. 카운트다운 실시간 감소. 종료 시 자동 숨김.
쿠폰적용가 활성 쿠폰 존재 시에만 노출. 쿠폰 만료 후 자동 숨김. 타임세일과 독립적으로 동작 가능.
"내일 비싸져요" 타임세일 종료까지 24시간 이내일 때 노출. 금액은 original_price - sale_price.
에디터 코멘트 상품 DB에 editor_comment 필드 존재 시 노출. 현재 우선순위 낮음 (기본 OFF).
후기 Empty State review_count == 0일 때 별점 분포 대신 "첫 후기 작성하기" empty state 노출. 별점 행도 빈 별(☆)로 전환.
무료배송 여부 상품의 is_free_shipping 기준. 무료 시 세이지 "무료배송" 뱃지, 유료 시 그레이 "배송비 N원" 뱃지 + "N원 더 담으면 무료배송" 임박 안내.
적립 포인트 point_rate 설정 시 노출. "구매 시 NP 적립". 포인트 제도 운영 여부에 따라 전역 ON/OFF.

※ 상단 토글(타임세일 · 에디터 · 후기 · 무료배송 · 적립)은 프로토타입 검토용. 실제 노출 여부는 위 표의 데이터 조건에 따라 자동 결정됨.

3. 샵바이 Enterprise 연동

샵바이 Enterprise 헤드리스 모드 기반. PDP 프론트는 자체 구축, 샵바이 API로 데이터 호출.

필수 API
· GET /products/{productNo} — 상품 기본 정보 (이름, 가격, 옵션, 이미지)
· GET /products/{productNo}/options — 옵션 목록
· GET /products/{productNo}/reviews — 후기 목록
· POST /cart — 장바구니 담기
· POST /orders — 바로 구매

자체 API (졸리팟 백엔드)
· GET /recipes?ingredient_id={id} — 연결 레시피 목록 (스타일링샷 데이터)
· GET /products/{id}/editor-comment — 에디터 코멘트
· GET /products/{id}/recommendations — 추천 상품

4. 스타일링샷 데이터 소스

P1 (현재): 상품-레시피 매핑 테이블에서 연결된 레시피의 썸네일을 가져옴. 썸네일 = 유튜브 영상 썸네일 (완성 요리 컷이 대부분).

P2 이후: Gemini Flash로 영상 마지막 프레임(완성 컷) 자동 추출 → 썸네일 대체. 더 감성적인 완성 사진으로 교체 가능.

카드 구조: 3:4 비율, 72% 폭, 횡스크롤 snap. 최소 2장 이상일 때만 섹션 노출. 1장이면 숨김.

판매가 24px / 700 / #2A2A2A (차콜)
쿠폰적용가 24px / 700 / #C75D3E (테라코타). 활성 쿠폰 있을 때만.
긴급성 뱃지 인라인 뱃지. 배경 rgba(214,69,69,.06) / 보더 rgba(214,69,69,.12) / 텍스트 #D64545

6. 스크롤 연동

스티키 탭: 네비바(46px) 아래에 고정 (top:46px). 3탭(상품정보 / 후기 / 문의) 스크롤 위치에 따라 활성 탭 자동 전환 (IntersectionObserver 또는 scroll event).

탭 클릭: 해당 섹션의 sec-anchor로 smooth scroll. scroll-margin-top:95px (네비+스티키 높이).

CTA 바: position:sticky; bottom:0. 항상 하단 고정.

7. 바텀시트 동작

"구매하기" 탭 → 오버레이 + 바텀시트 slide up.
오버레이 탭 또는 X 버튼 → 닫힘.
옵션 미선택 시 "장바구니 담기" / "바로 구매" 비활성 처리 필요 (프로토타입 미구현).
수량 최소 1, 최대는 재고 수량.
총 상품금액 = 옵션가 × 수량 (실시간 업데이트).

8. 디자인 토큰

Jollypot_Design_Guide.md 참조. 핵심 토큰:

· CTA: #C75D3E (테라코타)
· 서브: #8FA68E (세이지) — 상품정보 더보기 버튼
· 텍스트: #2A2A2A / #6B6B6B / #A8A8A8 (3단계)
· 카드: #F5F0E8 배경 + 1px #EDE8E0 보더 + 16px radius
· 순검정(#000) 사용 금지. 차콜(#2A2A2A) 통일.
· font-weight 700은 섹션 헤더(L1·L2)와 가격에만 사용.

9. 주의사항

· 쿠팡 파트너스 연동 상품은 쿠팡 링크 사용. 졸리팟 자체 상품(샵바이)과 쿠팡 파트너스 상품의 CTA 동작이 다름 — 쿠팡 상품은 "구매하기" 클릭 시 쿠팡 앱/웹으로 이동, 바텀시트 미사용.
· 스타일링샷의 레시피 카드 탭 → 해당 레시피 상세 페이지로 이동 (앱 내 화면 전환).
· 찜 토글 상태는 로컬 + 서버 동기화. 비로그인 시 로그인 유도.
· 선물하기는 샵바이 선물하기 API 연동. 미지원 상품은 아이콘 숨김 처리.
· 상품정보 더보기: 접힌 상태에서 max-height 720px → 펼치면 max-height 제거 + 버튼 "접기"로 전환.
· 후기 "리뷰 더보기" → 후기 전체 목록 페이지로 이동 (별도 화면).
· 포토리뷰 필터: API에서 has_image=true 파라미터로 필터링.
