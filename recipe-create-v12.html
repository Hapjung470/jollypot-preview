📋 개발 지침 — 만들기 플로우 착지/동선

  • 직접 작성 발행하기 → 방금 만든 레시피 상세(Case2 오리지널)로 착지 + "발행됐어요" 배너. 고노력 창작이라 결과물을 풀로 보여주는 게 보상. 스크랩북엔 자동 저장됨. (AI 경로와 달리 즉시 완성이라 "정리 중" 단계 없음)
  • AI 큐레이션 링크 → 컬렉션 선택(분석 백그라운드 병렬) → 레시피 담기 → 스크랩북 착지. 영상 메타(썸네일·제목·채널)는 즉시, 분석 결과(재료·시간)는 2~5분 비동기. 영상 카드에 "분석 중" 표시 동반.
  • 준비중 카드 담기 직후 스크랩북 그리드 맨 앞에 크림 슬롯 + 🥚정리중 + 출처뱃지(썸네일 없음, 출처 무관 통일). 완성 시 제자리 변신(썸네일+⏱시간) + 완료 푸시 알림. 빈 구간 동안 진행 가시화로 연속 담기 안심.
  • 닫기/건너뛰기 컬렉션 선택의 ✕·건너뛰기는 취소가 아님(분석 이미 시작). 최근 저장에 담고 스크랩북으로. ✕는 "최근 저장에 담았어요" 안내.
  • 담기 모델 컬렉션 탭 = 선택만(즉시 저장 아님), "레시피 담기" 버튼 = 확정 + 분석 트리거. 위치 안내는 버튼 아래 보조문구로(버튼 텍스트는 고정).
  • 에러 타이밍 즉시(입력 시)=미지원·비공개·길이초과 → 컬렉션 선택 전 차단. 분석 후(2~5분)=못 찾음·저신뢰 → 준비중 카드 3-way 해소(완성/확인필요/못찾음)+알림.
  • 에러 톤 가짜 레시피 안 만듦(못 찾으면 솔직히). 사용자 탓 X. 막다른 길 X. 길이초과=하드차단 아닌 "앞부분만" 대안. 저신뢰=불확실 필드만 노랑 확인(전체 편집 강요 X)+AI라벨·칼로리±20%·신고.
  • 미구현 이미 등록된 레시피 "레시피 바로가기" 다음 화면(별도 진행). 발견 크레딧(🥚 첫 큐레이터)·원작자 인증 프롬프트(P4 씨앗) 미반영. 종착→다음행동(요리시작→쿡모드 / 완성카드 탭→AI상세 Case1) 미연결.
