개발 참고 지침 (Development Notes)

1. 데모 전용 컴포넌트 (실 앱 제외)

  • 상단 노란 devbar: 시점·커버·Identity·그리드비율 토글은 디자인 비교용
  • DOMContentLoaded에서 그리드 셀 30개 동적 생성: 실 앱은 컬렉션 데이터 기반 렌더
  • 이 dev-notes 블록 자체

2. 시점 분기 (body class)

  • viewer-owner / viewer-guest: 사용자 권한 데이터로 결정
  • .owner-only-row / .guest-only: 시점에 따라 자동 표시·숨김

3. 커버 케이스 (body class — 컬렉션 콘텐츠 구성으로 자동 결정)

  • 숏폼 3+ → cover-shorts3 (3열 그리드 + "+N more" 더보기 슬롯)
  • 숏폼 2 → cover-shorts2 (좌정렬 2칸)
  • 숏폼 1 → cover-shorts1 (1칸)
  • 숏폼 0 (롱폼만) → cover-longform (16:9 풀블리드 폴백)
  • .cover-case-* class는 메인 커버 + 첫 추천 카드 미니커버에 공통 적용

4. 확정 사양

  • 요약 카드 영상 썸네일: M(125px) · 3:4 · 메타 오버레이 ON (갤럭시 S24 기준 5스텝 한 화면 진입)
  • 목록 그리드 셀: 3:4 · 3열
  • 커버 그리드: 9:16 · 3열 (역할 분리: 표지=임팩트, 목록=스캔)
  • 메인 CTA: 테라코타 #C75D3E
  • 노른자 #FDB813은 브랜드 IP 전용 — CTA 사용 금지

5. 게스트 인터랙션

  • 상단 [컬렉션 저장] 풀폭 CTA (탭 시 흰배경+세이지보더 완료 상태)
  • 하단 social-bar (RED 패턴): 댓글 입력 필드 + ♥·🔖·💬·↗
  • 스크롤 시 숨김 / 멈춤 시 등장 (debounce 200ms)
  • 모든 카운트 1+ 표기 (P1 — 임계값 미적용)

6. 오너 인터랙션

  • 상단 [편집][레시피 추가] 50:50
  • ⋯ 메뉴: [컬렉션 삭제]만 (P1 스코프 — 레시피 순서 변경 제외)
  • 편집 모달: 표지 자동/직접 토글, 컬렉션 소개, 공개 범위

7. 큐레이션 코멘트

  • .identity-desc ↔ 편집 모달 #descInput 동기화 필요 (저장 시 양쪽 갱신)
  • 입력 가이드: 만든 이유 + 추천 대상 (힌트 텍스트 참고)
  • 좋은 코멘트 = 좋은 인플루언서 선정의 신호

8. 추천 컬렉션 영역

  • 큐레이터의 다른 컬렉션: 세로 최대 2개 + 전체보기
  • 비슷한 컬렉션: 세로 무한 스크롤 (알고리즘 추천)
  • 각 카드 미니커버는 자기 콘텐츠 구성대로 렌더 (첫 카드만 데모용으로 메인 커버 case 추종)

9. 디자인 시스템 준수

  • 컬러·타이포 전반 Jollypot_Design_Guide.md v1.1 준수
  • 3단계 면 위계: #FFFFFF → 카드 #F5F0E8 + 1px #EDE8E0 → 강조 #F7F2EA
  • 그림자 사용 안 함 (플랫 + 보더)
  • 본문 텍스트 #2A2A2A (순검정 #000 금지)

10. 보류 · 후속 처리

  • 레시피 순서 변경 → P2 이후 (DEC-?, 평균 5-10개라 가치 약함 판단)
  • 큐레이터 차단 → profile-other 페이지로 이관 예정
  • 카운트 임계값 정책 → 트래픽 데이터 보고 재검토
  • 쿠팡 파트너스 법무 2건: disclaimer 시제 / 통신판매중개업 A12
