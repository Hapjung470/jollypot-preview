냉장고 사진을 ‘재고 스캐너’가 아니라 ‘빠른 입력 보조’로

“냉장고를 찍으면 다 정리된다”는 지킬 수 없는 약속입니다(통 안·가려진 재료는 사진에 정보가 없음). 그래서 사진을 완성 도구가 아니라, 수동 입력을 빠르게 해주는 보조로 재정의하고 — 냉장고 사진 + 영수증 + 바코드 여러 입력을 하나의 ‘확인 시트’로 모으는 안을 제안합니다.

← 목록으로
✅ 업데이트 — ‘냉장고 사진’ 보조 경로가 프로토타입에 반영됨
본 제안의 냉장고 사진 보조(보이는 것만 · precision 우선 · 별도 스캔실패 화면 없음 · 결과를 추가 시트에 미리 채움)가 fridge-v3-1-1 프로토타입에 구현됐습니다. 이 문서는 그 설계 근거와, 아직 남은 확장 후보(영수증 OCR·바코드)를 담은 분석본으로 보시면 됩니다.
한 줄 결론

① 사진 = 보조(assist), 권한(authority) 아님. AI 결과는 항상 확인 시트에서 사용자가 승인한 뒤 담깁니다(자동 저장 금지). 못 본 건 “원래 못 본다”고 정직하게 말합니다.

② 단일 소스(냉장고 사진)보다 멀티 소스가 정답. 특히 영수증 OCR은 텍스트라 정확도 95%+ — 냉장고 내부 사진(62~89%, 통 속은 0%)보다 높고, ‘장 본 직후 채우는 순간’에 딱 맞습니다.

③ 모델은 기존 Gemini Flash 재활용 — 새 비전 벤더 결정 없이 증분 개발. 수동(이번)이 먼저 나가고, 사진 보조는 그다음 단계(Phase 2)로 얹습니다.

✓ 채택: 냉장고 사진(보이는 것) ✓ 채택·강추: 영수증 OCR ✓ 채택: 단일 확인 시트 ✕ 기각: 전체 자동 스캔 ↦ 추후: 바코드 정밀추가
1. 왜 ‘냉장고 사진 = 재고 완성’이 안 되나 · 데이터 근거

회원님의 우려(입체·반찬통)는 현재 연구·상용 데이터로 그대로 확인됩니다. 모델을 더 키워도 못 푸는 ‘정보 부재’ 문제라서, 해법은 모델이 아니라 설계(기대치·입력 경로)에 있습니다.

측정된 한계

  • 가림(occlusion)이 1순위 난제. 사진은 앞줄 한 겹만 봅니다 — 뒤·아래는 프레임에 정보가 없어 추론하면 곧 환각이 됩니다.
  • 정확도가 종류에 크게 좌우. 포장·라벨 제품은 ~89%지만, 겹친 생채소는 62~74%로 떨어집니다(조명·각도까지 겹치면 더).
  • 반찬통/밀폐용기 = 인식 불가. 통 안 내용물은 라벨이 없어 어떤 비전 모델도 못 읽습니다. 한국 냉장고 핵심 비중이 바로 여기입니다.
그래서 ‘완성형 스캔’은 기각
“찍으면 정리됨”을 내세우면 첫 시도에서 기대-결과 격차로 신뢰가 깨집니다. 냉장고 기능은 첫인상에서 한 번 실망하면 재방문이 끊깁니다. → 약속을 ‘보이는 것만’으로 좁히는 것이 핵심 설계 결정.
2. 설계 원칙 5 · 보조로서의 사진
P1
정직한 범위
“보이는 것만 빠르게 담아요. 통 안·가려진 건 직접 추가.” 완전성을 약속하지 않음 → 실망 차단.
P2
항상 확인
AI는 초안만. 확인 시트에서 승인해야 담김. 자동 커밋 금지(human-in-the-loop).
P3
막다른 길 없음
인식 0/저조여도 ‘실패’ 벽이 아니라 수동 빠른추가로 매끄럽게 폴백.
P4
저마찰 병합
인식 결과 = 미리 체크된 칩. 위치·날짜 기본값. 한 번에 N개 담기.
P5
중복 제거
이미 있는 재료는 ‘이미 있음’ 표기·기본 제외. 정규화 이름으로 대조.
3. 캡처 소스 비교 분석 · 무엇으로 ‘찍을’ 것인가

“사진”을 냉장고 내부로만 좁히면 약점이 큽니다. 입력 경로를 늘리되 결과는 한 시트로 모읍니다.

입력 방식정확도커버리지맞는 순간비용/난이도판정
A. 전체 자동 스캔
“냉장고 다 인식”		낮음·불안정겉보기 전체(허상)없음높음(비전 풀)✕ 기각
B. 냉장고 사진
보이는 것만		포장89%/채소62~74%앞줄·포장·생채소채워둔 냉장고 1차 등록중(Gemini 재활용)✓ 채택
C. 영수증 OCR
방금 산 것		95%+(텍스트)구매 품목 전부장 본 직후 채우기중(OCR+매핑)✓ 강추
D. 바코드
포장 단품		매우 높음(단품)포장식품만단품 정밀 추가중(상품DB)↦ 추후
핵심 인사이트 — 영수증이 ‘냉장고 사진’보다 낫다
냉장고 내부 사진의 약점(가림·통)은 ‘냉장고를 찍는다’는 전제 자체에서 옵니다. 그런데 사용자가 재료를 넣는 순간 = 장을 본 직후이고, 그때 영수증(텍스트)이 손에 있습니다. 영수증은 가림·용기 문제가 없고 정확도도 높아, “넣을 때 빠르게”라는 본래 목적에 더 정확히 맞습니다. (상용 팬트리 앱들이 영수증→정규화→중복병합→보관기한 기본값을 이미 표준 패턴으로 사용)
4. 최적안 · 멀티 소스 → 단일 확인 시트

진입은 여러 갈래(냉장고 사진·영수증·바코드)지만, 모두 같은 ‘확인하고 담기’ 시트로 합류합니다. 사용자는 한 가지 확인 동작만 배우면 되고, 수동 입력과도 자연스럽게 섞입니다.

흐름 한눈에

진입: 빈 상태 화면 / 양문형 ‘＋추가’ 시트 상단에 “📷 사진으로 빠르게 추가” 보조 버튼. 누르면 소스 선택(냉장고 안 · 영수증 · 바코드[추후]).
처리: 촬영/선택 → “보이는 재료를 찾는 중…”(짧게). 마이크로카피로 한계를 미리 고지(“통 안·가려진 건 못 봐요”).
확인 시트(핵심): 인식 결과를 편집 가능한 체크리스트로. 항목별 이름·위치(냉동/냉장)·신뢰도. 고신뢰=미리 체크, 저신뢰=‘확인’ 점선. 이미 있음 자동 표기. 날짜 기본 ‘오늘’. → [N개 담기].
결과: 양문형에 일괄 반영 + 토스트. 저신뢰 항목만 가볍게 재확인 유도.
폴백: 또렷한 게 없으면 ‘실패’ 대신 수동 빠른추가 칩을 바로 노출(막다른 길 0).

6. 백엔드 · 데이터 스펙 · 개발사 핸드오프

모델 & 비용

  • 모델 재활용: 인스타 레시피 추출과 동일한 Gemini Flash(멀티모달) 사용 → 새 비전 벤더 결정·신규 통합 없음(이전 AI안의 ‘견적 대부분’ 리스크가 여기서 사라짐).
  • 영수증/바코드: 영수증은 동일 멀티모달 LLM의 OCR+구조화로 처리 가능(별도 OCR SDK 선택은 옵션). 바코드는 추후 상품 DB 연동 단계.
  • 비용 제어: 1회 1장, 클라이언트 다운스케일 후 전송. 사진은 일시 처리·미저장(프라이버시 고지).
인식 출력 계약 (확인 시트로 들어오는 형태) — 결정론 규칙으로 게이팅
items[] = {
  name,              // 한국 표준 식재료명으로 정규화 (약어·외래어 매핑: "ORG시금치"→"시금치")
  location_guess,    // "냉장" | "냉동" (기본 추정, 사용자 변경 가능)
  confidence,        // 0~1 — 임계 이상=미리 체크, 미만=‘확인’ 점선·미체크
  quantity?,         // 영수증/바코드만 (사진은 수량 추측 금지)
  source             // "fridge_photo" | "receipt" | "barcode"
}
규칙: ① 명확히 보이는 것만 반환 ② 통/가림 내용물 추론 금지(반환 안 함)
      ③ 수량·신선도 임의 생성 금지 ④ 비식품 제외 ⑤ 자동 저장 금지(항상 확인 시트 경유)
중복 제거 · 신뢰도 처리 (확인 시트 로직)
중복: 현재 냉장고 재료와 정규화 이름으로 대조 → ‘이미 있음’ 표기·기본 제외(영수증은 ‘수량 합산’ 옵션). 신뢰도: 고신뢰=미리 체크 / 저신뢰=미체크+‘확인’ 플래그. 저신뢰는 절대 자동 반영 안 함(잘못 담기보다 빠뜨리기를 택함).
7. 단계(Phasing) & 견적 영향
Phase 1 · 이번(확정)
수동 입력
빠른추가 칩 · 직접입력 · 양문형 관리 · 레시피 매칭. 사진 없이도 완결되는 기준선.
Phase 2 · 다음(핏 확인 후)
📷 사진 보조
냉장고 사진 + 영수증 OCR을 같은 확인 시트로. Gemini 재활용이라 증분 개발. 본 문서 범위.
Phase 3 · 추후
바코드 정밀추가
포장 단품용. 상품 DB 연동 필요. 정확도 최상이나 커버리지 좁음 → 마지막.
견적 관점 요약
사진 보조는 수동을 대체하지 않고 가속합니다(수동이 항상 폴백). 새 모델 결정이 없어 이전 AI 비전안보다 작고 안전한 증분이며, “핏을 본 뒤 얹는다”는 린 원칙에 정확히 맞습니다. 가장 비용 대비 효과가 큰 1순위는 ‘영수증 OCR’(정확도·맞는 순간 모두 우위)을 권장합니다.
8. 정직한 한계 & 리스크
  • 사진 경로는 여전히 부분적. 그래서 ‘완성’이 아니라 ‘보조’로 포지셔닝 — 확인 시트가 안전망.
  • 영수증은 마트별 포맷 편차가 있으나, 텍스트라 3D 장면보다 훨씬 다루기 쉬움(약어→표준명 매핑 사전 필요).
  • 사진을 ‘기본·완전’으로 오인하지 않게. 수동을 항상 1차로 두고, 사진은 “빠르게(보이는 것)”로 라벨링.
  • 진짜 성공지표는 정확도가 아니라 리텐션(추가 후에도 계속 쓰는가). 보조 입력은 ‘첫 등록 마찰’을 줄여 리텐션에 기여하는 수단으로 평가.
