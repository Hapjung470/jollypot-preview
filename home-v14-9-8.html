홈 v14-2 — 데크 CTA에 만드는 법 단계수 표기

v14-2 · "만드는 법 N단계"데크 1차 CTA에 실제 단계수 노출(만드는 법 10단계 / 8단계 / 5단계 …). 재료 미리보기에 이어 "손 가는 정도"를 탭 전에 신호 — ⏱시간(소요)과 별개로 난이도/공수 감을 줘서, 저장만 하고 안 만드는 이탈을 줄이고 자기 상황에 맞는 레시피 선택을 도움. 단계수는 레시피 데이터에서 자동 산출.

— v14-1 데크 CTA 변경 —

홈 v14-1 — 데크 1차 CTA 변경 (요리 시작 → 만드는 법)

v14-1 · 데크 CTA = 만드는 법(→ 상세)슬림 데크의 1차 CTA를 "요리 시작"에서 "만드는 법"으로. 근거: ① 데크는 발견 표면 — 사용자는 "볼까" 단계지 "지금 따라 만들자"가 아님 ② 재료만 본 상태에서 쿡모드(전체화면 따라하기) 직행은 이르다, 스텝 훑고 결정해야 함 ③ 요리 시작은 사라지는 게 아니라 상세 안의 1차 CTA로 이동 — 깔때기: 데크 →(만드는 법)→ 상세 →[장보기 / 요리 시작] ④ 전략적으로 커머스 진입이 "상세→장보기"라, 상세로 유입시켜야 함(쿡모드 직행은 커머스 스킵). "요리 시작"의 1차 CTA 자리 = 상세·스크랩북·최근 저장(이미 결정한 표면).

— v14 결정 확정 —

홈 v14 — 결정 확정 (데크 슬림 · 퀵필터 제거)

v14 · 데크 = 결정카드(슬림) 정식 채택 [DEC 등록 필요]풀 레시피 통째 → 썸네일+제목+출처+메타+재료+[저장][요리 시작]로 확정. 만드는 법은 상세/쿡모드로 이전. 근거: 발견 단계엔 실행 디테일(10스텝) 불필요, 첫 화면 레시피 1개 독점 해소, 카드 높이 ~45%↓. 꺼내쓰기는 "요리 시작 1탭"이 보장. 풀레시피는 상단 토글에 비교용("이전")으로만 남김 — 기본·현재는 슬림.

v14 · 퀵필터 제거 [DEC 등록 필요]제약 기반 칩 행 철회 — "핵심이 아니다" 판단. 데크가 이미 "뭐 해주지"의 답을 결정적으로 주므로, 추가 능동 진입 레이어는 홈의 초점을 분산. 발견 보강은 검색·전체 카테고리로 충분. CSS·HTML·JS·프로토 토글 전부 제거.

v14 · 후속 (별건)① 큐레이션 룰: "오늘 저녁" 헤더엔 저녁 메인이 뜨게(현재 데크에 케이크·팬케이크 섞임) — 콘텐츠 큐레이션 작업, UI와 별개. ② 데크 개인화(기존 유저 저장 이력 반영) 여전히 오픈.

— v13-1 검토 패치 (조엘 피드백 01·02·03) —

v13-1 · 01 데크 재검토 → 풀↔슬림 토글데크 개념(큐레이션 결정 카드 스와이프)은 유지. 다만 풀 레시피 통째는 무겁다는 판단: ① 접힌 상태도 카드 ≈600px라 첫 화면을 레시피 1개가 독점 ② "오늘 저녁" 헤더 vs 데크 내용(케이크·팬케이크·콩국수) 불일치 ③ 만드는 법 10스텝은 "할까 말까" 결정엔 불필요한 실행 정보. 결정카드(슬림) = 썸네일+제목+출처+메타+재료+[저장][요리 시작], 만드는 법은 상세/쿡모드로. 높이 ~40%↓. 꺼내쓰기는 "요리 시작 1탭"이 보장. 프로토 토글로 풀↔슬림 직접 비교. (헤더↔내용 불일치는 큐레이션 룰 문제라 별도 — 저녁엔 저녁 메인이 뜨게)

v13-1 · 02 퀵필터 맥락 적응형"항상 같은 칩 = 배너 블라인드" 지적 수용. 비고정(스크롤 시 사라짐)은 이미 됐고, 진짜 문제인 내용 고정을 해소: 시간대·요일로 칩 세트 변경(평일 저녁 / 주말 / 평일 아침 / 야식). 접속 시간 자동 + 프로토 토글로 세트 미리보기. 다음 단계는 저장 이력 반영(자주 저장한 카테고리 우선).

v13-1 · 03 피드 출처점 롤백"크리에이터 표기는 레시피 상세에 있어 문제없음" 판단 수용. 피드 카드 = 크리에이터명만(v12.13.1 상태 복귀), 데크의 claimed/unclaimed 점 통일 철회. 잔여: 피드만 훑는 유저는 외부 출처를 졸리팟 크리에이터로 볼 여지 — 가입 크리에이터 증가 시 재검토 포인트로만 보류.

— v13 최적안 (스코어링 반영) —

v13 · ① 퀵필터 행 (탐색 5/10 보강)헤더 바로 아래에 제약 기반 칩 행(20분 이내·냉장고 털기·아이 반찬·가볍게·한 그릇·전체). "오늘 뭐 해주지"의 능동 경로 제공 — 기존엔 검색 1개 + 수동 캐러셀뿐이라 "지금 닭 있는데 20분 안에"식 의도를 즉시 받아줄 길이 없었음. 비고정(디자인 가이드 non-sticky) + 세이지 분류 톤(테라코타 침범 금지).

v13 · ② Peak-End 요약 카드 복원 (마무리 4.5/10 보강)스펙엔 있는데 실제 렌더가 빠져 있던 "오늘 N개 저장했어요" 카드 복원. 9섹션 끝 → 무한그리드 직전에 배치. 저장할 때마다 실시간 카운트(데크/압축/더블탭/무한 전부 연동) + 스크랩북 진입 CTA. 0개일 땐 "저장한 레시피는 스크랩북에 모여요"로 안내, 1개+부터 카운트. 흐지부지하던 끝에 보상+전환 고리 복구.

v13 · ③ 출처점 통일 (정보 일관성 위반 수정)데크는 claimed(세이지 점)·unclaimed(회색 점+YouTube에서 보기)을 정확히 구분하는데, 압축·무한 카드는 백종원·이연복을 그냥 크리에이터처럼 표기(DEC-023·v12.5.1 원칙 위반, 미가입자 오인 리스크). 전 피드 카드에 데크와 같은 점 언어 적용: 가입 크리에이터 = 세이지 점+이름 / 외부 영상 = 회색 점+"유튜브 레시피"(개인명 미표기). 일관성+신뢰 회복.

v13 · ④ 시간대 적응 헤더데크 헤더 "오늘 저녁 뭐 해주지?"를 접속 시간대별로(아침/점심/저녁/야식) 자동 전환. "뭐 해주지?" 브랜드 프레임은 유지. 점심에 접속해도 "저녁"이라 뜨던 어긋남 해소(저비용·고관련성).

v13 · ⑤ 전체 카테고리 보기 (릴리프 밸브)16→9 축소 때 빠진 테마(야식·비건·다이어트·명절 등)로 가는 출구. 의도는 있었으나 body에 미렌더였던 요소 추가. 9섹션 끝에 배치해 "여기 없으면 여기로" 신호.

v13 · 미반영 — 네 결정 필요 (구조 의사결정 단위)① 캐러셀 월 분해: 데크 아래 9개 연속 횡스크롤은 리듬 단조+횡스크롤 깊은 항목 미노출. 강 섹션 4~5개만 남기고 나머지를 무한 세로피드로 조기 전환하는 게 다음 큰 레버. ② 데크 개인화: 기존 유저 히어로(가장 비싼 자리)가 전원 동일 에디터 픽 — 저장 이력 반영 데크가 리텐션 핵심. 둘 다 서버/구조 변경이라 임의 반영 않고 분리.

— 이전 변경 이력 (v12.x) —

v12.6.1 · "최근에 저장한 레시피" (라벨·구조 정리)① 라벨 수정: "이어서 만들기"는 진행 중 재개 뉘앙스라 부적합 → "최근에 저장한 레시피 / 오늘 만들어보실래요?". 스크랩북 Tier1(최근 저장)과 라벨 일관. ② 구조 통일: 딥크림 배경 박스 + 16:9 가로 썸네일 → 배경 제거 + 3:4 세로 카드로 16섹션 압축카드와 완전 동일. 저장됨 표시(채운 북마크)만 우하단 유지.

v12.6 · C 16섹션 → 9섹션 압축결정피로 해소. 16개는 선택지 과잉이라 "둘러보기"로 회귀 위험. 핵심 9개(화제·5분·키즈·제철·김치·아이·가볍게·디저트·도시락)만 몰입 흐름으로 재배치. 나머지는 "전체 카테고리 보기"로.

v12.6 · C 끝 요약 카드 (Peak-End)카탈로그 끝에 "오늘 N개 저장했어요" 카드. 저장할 때마다 카운트 실시간 증가(직접 저장해보면 작동). 흐지부지하던 끝을 마무리 보상 + 스크랩북 진입으로 전환.

v12.13.1 · 커머스 토글 비교상단 컨트롤바에 커머스 토글 추가. "상품 제외·현재"(기본) ↔ "상품 포함·이전"(키즈 셀렉션 3번 + 가볍게 7번 원위치 복원)을 즉석 전환해 before/after 비교. 대표 설득용 — "뺐더니 깔끔해졌다"를 한 화면에서 시연.

v12.13 · 홈 상품 섹션 제거스타셸 대표 피드백 수용: 홈의 상품 섹션 2개(키즈 셀렉션·가볍게 먹는) 제거 → 콘텐츠 7섹션 홈. 이질감의 정체는 "커머스"가 아니라 "맥락 없는 커머스". 홈은 레시피 발견·저장에 집중, 커머스 진입은 레시피 상세→장보기로(명제 "그 다음"). 영구 제거 아님 — 라이프 솔루션 형태로 복귀 예정(DEC-027).

v12.12.3 · 진행바 클릭 점프데크 진행바를 누르면 해당 카드로 점프(인스타 스토리식). 3px라 클릭이 어려워 위아래 7px 히트영역 확대(시각 막대는 ::after로 3px 유지). 진행바를 button으로 전환 + carousel.scrollTo로 해당 슬롯 가로 스크롤(세로 영향 없음). 1~5번 어디든 한 번에 이동.

v12.12.2 · ① 메타 한 줄데크 메타가 두 줄로 넘치던 건 이모지가 아니라 flex-wrap: wrap 때문. 이모지 유지(시간/인분/칼로리를 글자 없이 구분 = 정보 밀도 좋음)하고 nowrap + gap 축소로 한 줄 고정.

v12.12.1 · 데크 스와이프 수정오른쪽 스와이프 안 되던 문제: ① scroll-padding-left(carousel)와 슬롯 padding-left 중복으로 snap 위치 어긋남 → scroll-padding 제거 ② scroll-snap-stop: always 과도한 제약 제거 ③ 데스크탑 마우스 드래그 스크롤 JS 추가 (artifact 뷰어는 데스크탑이라 마우스 드래그가 기본 안 됨 — 이게 주원인일 가능성). 드래그 중 snap 일시 해제, 놓으면 복원. 드래그 후 클릭 오발동 방지.

v12.12 · 데크 메타 칩리스 이모지데크 카드 메타(시간·인분·칼로리)에서 흰 배경 칩 컨테이너 제거 → 이모지 구분 텍스트 한 줄. ⏱ 24분 · 🍽 4인분 · 🔥 180kcal. 썸네일 하단 라인에 맞춰 왼쪽 정렬. 칩이 사라져 시각 노이즈 ↓, 정보는 그대로. 시간 이모지(⏱)는 9섹션·무한그리드 시간칩과 일관.

v12.11 · 인사 제거 + 메인 카피 맥락 헤더"안녕하세요 조엘님" 제거. 이름 단독은 최상단(가장 비싼 자리)에서 정보 0 + 재방문 시 노이즈 + 데크 헤더와 중복. 대신 데크 헤더를 메인 카피 "오늘 저녁 뭐 해주지?"로 교체 → 첫 줄이 곧 맥락 헤더. 사용자가 온 이유(저녁 고민)를 즉시 받아주고, 데크 5장이 그 답이 됨. 배민식 "의도를 건드리는" 진입.

v12.10 · ① 데크 Peek데크 카드 폭을 살짝 줄여(calc(100%-40px)) 다음 카드 가장자리 ~20px가 보이게. "옆에 더 있다"가 영구 어포던스로 전달돼 스와이프 학습 불필요. scroll-padding-left로 첫 카드 정렬 유지. Apple/Spotify 캐러셀 표준.

v12.9.1 · 최근저장 카드 규격 통일최근 저장 카드도 동일 규격으로: 시간칩(⏱ 좌하단) + 제목 2줄 + 1줄 메타. 단 메타 자리에 크리에이터 대신 "3일 전 저장" 유지 — "내가 저장한 것"이라는 정체성 신호. 저장됨 북마크(채움)도 우하단 유지. 규격은 통일, 정체성은 보존.

v12.9 · 9섹션 카드 규격 통일9섹션 횡스크롤 카드를 무한 그리드 카드와 동일 규격으로: 3:4 썸네일 + 시간칩(⏱ 좌하단) + 제목 2줄 + 크리에이터 1줄. 기존 "20분 · 재료 7개" 메타 → 크리에이터 이름으로 교체(스크랩북·무한그리드와 일관). 홈 전체 레시피 카드가 단일 컴포넌트로 통일됨.

v12.8 · 무한 로딩 "이런 레시피는 어때요?"9섹션을 다 보면 2단 그리드(3:4 세로)가 무한 로딩. 배민 맛집 탐색처럼 "오늘 저녁 뭐 해주지" 고민하며 내리는 탐색에 대응. 안 내리는 사람은 9섹션으로 끝, 탐색하는 사람은 무한. 카드 = 스크랩북 레시피 탭 구조(시간칩 좌하단 + 제목 2줄 + 크리에이터 1줄) + 저장 북마크. IntersectionObserver로 끝 감지 → 12장씩(데모 60장 후 종료). 끝까지 스크롤하면 작동.

v12.5.4 · 오집 북마크 형태상단 사각 모서리도 라운딩(r2.5) 강화 + 하단 부드러운 곡선. 오집 북마크와 동일한 부드러운 리본형으로 통일.

v12.5.1 · 저장 아이콘 칩리스 (오집 스타일)저장 버튼에서 반투명 다크 칩 제거 → 외곽선 북마크 아이콘만 (흰색 + drop-shadow). 하단 그라데이션으로 가독성 확보. 저장 시 테라코타 채움. 상품 카드(흰 배경)는 어두운 아이콘 + 흰 그림자로 대응.

v12.5 · 실사 음식 이미지인스타 스크린샷에서 음식 15장 크롭(조회수 텍스트 제외). 압축 카드 56개 + 데크 4장 실사 적용. 메뉴-이미지는 실사 질감 placeholder(실제 앱은 영상 썸네일).

v12.4 · 저장 액션 (위치 = 우하단)발견 흐름에서 저장이 막혀 있던 문제 해결. ① 압축 카드: 썸네일 우하단 북마크 (반투명 다크 원형, 미저장 흰선 / 저장 테라코타 채움). 좌하단 조회수 ↔ 우하단 북마크 대칭. ② 상품 카드: 우하단 장바구니 (미담음 흰 배경 / 담음 테라코타). ③ 데크 카드: 크림 박스라 썸네일 없음 → 하단 액션 로우에 [저장][요리시작] 나란히 (저장 보조·좌측, 요리시작 주·우측). ④ 더블탭 보조: 압축 카드 본문 더블탭 = 저장 + 🔖 팝 애니메이션 (인스타 패턴). ⑤ 피드백: 아이콘 채움 + 스케일 펄스 + 첫 저장 시 1회 토스트. 1-탭 저장 보장, thumb zone(우하단) 최적.

분리 = 얇은 라인 (옵션 B, v12.3.3 확정)3가지(없음/얇은 라인/두꺼운 띠) 비교 후 얇은 라인 채택. 이유: ① 풀 요약 카드 자체가 크림+보더로 닫힌 박스라, 두꺼운 띠(크림)는 크림→크림 중복으로 분리감 약함 ② 데크(큰 카드 1장) → 16섹션(작은 카드 횡스크롤)의 리듬 변화가 이미 강한 분리 신호 ③ 흰 배경 위 1px은 카드와 색 안 겹치고 노이즈 없이 모드 전환만 신호.

진행바 농도 구분흰 동그라미 마커가 너무 강했음. 제거하고 opacity 차이로 자연스럽게 위계: passed = 세이지 35% opacity (이미 본 카드, 옅게) / active = 세이지 100% (현재 카드, 진하게) / future = 라인 색 (안 본 카드). 색 같지만 농도로 시선이 active에 집중.

수정 2 · 하단 도트 제거도트와 진행바는 같은 정보(5장 중 현재 위치) 중복 표시. 진행바만으로 충분 → 도트 nav 영역 통째 제거. 시각 노이즈 ↓, 데크 영역 컴팩트.
