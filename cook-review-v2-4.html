DEV 요리후기 상세 v2 — 설계 의도 & 규칙

후기 = 리뷰가 아니라 실행 증언(execution testimony). 페이지 척추: 증언 → 신뢰 → 재실행

① 증언 — 신뢰 칩 (작성 마찰 최소화가 핵심)

  • CLOSE 단계에서 원탭 칩으로 수집 (타이핑 0). 다차원 별점 금지 — 작성 리텐션 보호
  • 칩 그룹(원탭, 긍정 시그널 위주): 재현도(레시피대로 됐어요), 난이도 체감(생각보다 쉬웠어요 / 까다로웠어요), 팁(간 조절 필요 등)
  • 변형 한 줄 입력은 미채택 — 변형 내용은 후기 본문에 자유 기술. 작성 마찰 최소화 우선 (변형 callout/칩 제거)
  • 별점은 단일 5점 유지

② 신뢰 — 재현 신뢰 시그널

  • "이 레시피로 N번째 만든 요리" = 재실행 블록 라벨로 카드 위에 1개만 노출 (중복 제거: '이 레시피로 만들었어요' 라벨은 폐지하고 이 문구로 통합)
  • 작성자 신뢰 단서(단골 라벨/누적 요리 수)는 작성자 영역 근처 — P2

③ 재실행 — 적응형 CTA (가장 큰 레버리지: 꺼내쓰기 루프 닫기)

  • 식단에 추가는 미구현 — 식단(MealSlot) 기능 확정 후 본인 시점 보조 CTA로 추가 예정

DEV 엔게이지먼트 카운트 노출 규칙 (공통)

콘텐츠 상세 공통 엔게이지먼트 바 (레시피 · 컬렉션 · 요리후기)
  • 본인 시점 → 카운트 숫자 항상 미노출 (작성 리텐션 보호)
  • count = 0 → 숫자 미노출, 아이콘만 (공통 · 영구)
  • 0 < count < 임계값 → 숫자 미노출 (부정적 사회증명 차단)
  • count ≥ 임계값 → 숫자 노출
지표임계값근거
♡ 좋아요10가장 가벼운 참여 → 높게
🔖 저장5의도적 행동 → 중간
💬 댓글3가장 무거운 참여 → 낮게
const THRESHOLD = { like: 10, save: 5, comment: 3 };
function showCount(metric, count, isOwnerView) {
  if (isOwnerView) return false;
  if (count <= 0) return false;
  return count >= THRESHOLD[metric];
}
⚠️ 임계값은 출시 초기 부정적 사회증명 방지용 운영 파라미터. 콘텐츠 누적 후 상향/제거 재검토. (서버 config 권장)
